# MCPlexer v0.9.9

MCPlexer v0.9.9 is a public `AGPL-3.0-or-later` release. It hardens the adaptive
`mcpx__call_tool` path so agents pick the cheap direct call when it is safe,
and fall back to Code Mode when results would otherwise burn context.

## Highlights

- **`mcpx__call_tool` path guidance.** Tool description and harness
  instructions teach a clear rule of thumb: use `call_tool` only when the
  entire result is the answer; if you would map, filter, or pick fields, use
  `mcpx__execute_code` and `print` only what you need. Schema examples prefer
  small independent tools (for example `index__status`, `memory__save`) over
  fat hydrates.

- **Large direct-result advisory.** When a direct `call_tool` target returns a
  large envelope, the gateway adds a soft `_call_tool_hint` so the next call
  can switch to Code Mode + filtered `print` without reshaping the payload.

- **Clearer tool-not-found recovery.** Route misses and unknown built-in names
  surface did-you-mean suggestions (and a `mcpx__search_tools` pointer) so
  typos recover in one step.

- **Code Mode namespace-shadow lint.** Bindings that reuse a registered tool
  namespace (for example `const mesh = mesh.receive(...)`) emit a warning
  before the Temporal Dead Zone `ReferenceError`, naming the rename fix.

- **`task__get` preview by default.** Single-task reads return
  `task_view: "preview"`: note counts plus a few truncated previews and a
  status-history count, without dumping full note history. Pass `full: true`
  for the historical hydrate (all notes + full status history).

- **`using-mcplexer` skill contract.** The bootstrap skill documents the
  decision rule, good/bad examples, anti-patterns, and `_call_tool_hint`
  handling so every harness that fetches the skill gets the same path choice.

## Upgrade notes

- Back up the MCPlexer application state before upgrading. This release adds no
  new database migrations over v0.9.8.
- The daemon upgrade is an in-place atomic swap with drain-wait, readiness
  verification, and automatic rollback (`task upgrade`). Expect roughly one to
  two seconds of downtime.
- Agents that previously relied on `task__get` always returning full notes
  should pass `full: true` (or filter in Code Mode). Default preview is the
  intentional cheap path.
- Harness sessions started before the upgrade may need a reconnect to pick up
  updated tool descriptions and instructions.

## Verify a download

Verify the archive digest with `checksums.txt`, then verify its build identity:

```bash
gh attestation verify mcplexer_v0.9.9_linux_amd64.tar.gz \
  --repo Don-Works/mcplexer
```

The preferred source for this release is the `v0.9.9` tag and its public GitHub
source archive. The release workflow independently binds every artifact to the
tag's commit.
