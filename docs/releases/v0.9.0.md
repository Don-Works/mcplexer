# MCPlexer v0.9.0

MCPlexer v0.9.0 is a public `AGPL-3.0-or-later` release focused on reliable
delegation, durable monitoring, deterministic skill composition, and safer
execution boundaries.

## Highlights

- Delegation now performs preflight validation, injects repository context,
  enforces capacity, and ranks eligible models using observed quality and
  reliability.
- Monitoring incidents, occurrences, triage claims, and completion receipts
  are durable. Repeated events update one canonical incident instead of
  creating duplicate work.
- Skill-registry composition is deterministic and pin-aware, with scope-aware
  audits, atomic installation, deletion guards, and administrative inspection
  tools.
- Code Mode adds the `mcpx__retrieve` compression surface, tolerant discovery
  arguments, JavaScript aliases, and clearer lint/did-you-mean feedback.
- SQLite read concurrency and WAL handling were tightened, and the brain poller
  reacts to committed changes without holding a permanent read transaction.

## Security and release integrity

- Workspace ACL lookup, skill composition, and sandbox path masking now fail
  closed.
- Hook access, browser authentication, cookies, query tokens, and public error
  responses have narrower trust boundaries.
- Notification navigation rejects external, scheme-relative, and backslash
  paths; transient notification identities are collision-safe.
- The dashboard exposes the AGPL source, copyright, and no-warranty notice.
- Public-tree hygiene checks run in CI without printing configured sensitive
  patterns.
- CodeQL analyzes both the Go daemon and dashboard JavaScript on pushes,
  pull requests, and a weekly schedule.
- Model-provider subprocesses receive a strict environment allowlist, so
  unrelated integration credentials and future secret-bearing variables are
  not inherited accidentally.
- The optional P2P DHT now applies the upstream IP-diversity filter's
  recommended per-network limits. This mitigates Sybil concentration described
  by `GO-2024-3218`; the Go vulnerability record still lists every package
  version because it has no fixed-version or symbol metadata. See
  `SECURITY.md` for the applicability assessment.
- API-backed delegated repository work defaults to a unique owned git
  worktree. The gateway exposes only exact, relative-path workspace file
  operations inside that checkout; traversal, symlink escapes, Git metadata,
  and undeclared write paths fail closed.
- The isolation and persistence packages are tested natively on macOS and
  Windows before the release workflow can publish artifacts.
- All six platform archives are structurally inspected and checksum-verified.
  The matching native archive is also executed on each Linux, macOS, and
  Windows runner. Every archive includes exact third-party license metadata;
  the release also includes an SPDX SBOM and Sigstore-backed GitHub
  attestations for all six archive digests.

## Upgrade notes

- This release follows a one-time public-history hygiene rewrite. Existing
  contributors should make a separate patch backup of local work, reclone the
  repository, and reapply only reviewed patches. Old commit IDs and signatures
  are no longer authoritative; do not merge old refs back into rewritten
  history.
- Back up the MCPlexer application state before upgrading. Database migrations
  143 and 144 run automatically: 143 creates the durable monitoring incident
  tables, and 144 records trusted delegated-worktree result metadata.
- Existing log templates are marked as already triaged during migration so an
  upgrade does not wake workers for the full historical backlog.
- The release binary includes P2P support. Existing workspace ACLs and peer
  grants remain authoritative after upgrade.
- macOS subprocess isolation is deny-by-default. Supported model and LM Studio
  CLIs receive explicit executable, workspace, and minimum configuration paths;
  undeclared host paths remain inaccessible.
- CLI-backed model providers remain explicit opt-ins. They retain
  `NetworkHost` access so they can reach their configured provider; only enable
  them for trusted commands and endpoints.
- All worktree-isolated CLI delegates fail before checkout creation, secret
  resolution, or model launch. A CLI can otherwise connect directly to the
  main daemon socket and omit per-run claims. Use an API-backed delegate for
  worktree isolation; `worker_isolation: none` is the explicit trusted escape
  hatch until a capability-required per-run relay is available.
- Delegated worktree checkouts are temporary. The trusted runner snapshots
  validated changes after the post-execute hook, records the result commit and
  changed state, retains a result branch when changes or recovery require it,
  and cleans up only after terminal result persistence. Snapshot or persistence
  failures retain recovery evidence instead of discarding work.

## Verify a download

Verify the archive digest with `checksums.txt`, then verify its build identity:

```bash
gh attestation verify mcplexer_v0.9.0_linux_amd64.tar.gz \
  --repo Don-Works/mcplexer
```

The preferred source for this release is the `v0.9.0` tag and its public GitHub
source archive. The release operator creates and locally verifies the signed
tag before publication; the workflow independently binds every artifact to the
tag's commit.
